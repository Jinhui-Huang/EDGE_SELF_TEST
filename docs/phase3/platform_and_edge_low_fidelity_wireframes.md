**定位**: 面向企业级网页自动化测试平台的低保真页面线框图说明文档，覆盖 **后台主平台** 与 **Edge 插件辅助入口** 的核心页面布局、组件摆放、交互区域与信息层次。

- 给前端开发 AI / Claude Code / ChatGPT / 本地模型做页面开发输入

# 1. 文档目标

本文件不追求高保真 UI 视觉，而是用**文字框图 + 布局说明**定义：

- 页面区域如何分布
- 哪些信息放在哪里
- 哪些按钮/表单/表格应该出现
- 页面最基本的操作动线
- 后台主平台和 Edge 插件分别承担什么职责

---

# 2. 使用说明

## 2.1 低保真线框图是什么
低保真线框图是“页面结构草图”，主要表达：
- 布局
- 信息层级
- 功能入口
- 交互位置

它不是最终视觉稿，因此不强调：
- 颜色
- 字体
- 动效
- 图标风格

## 2.2 适合用来做什么
- 交给前端 AI 先出页面骨架
- 做 React/Vue 管理台第一版
- 做产品交互确认
- 让 UI 设计师继续往高保真设计发展

---

# 3. 设计约定

## 3.1 后台主平台布局约定
后台主平台统一使用：

- 顶部全局导航
- 左侧主菜单
- 中间主内容区
- 可选右侧详情/日志区

## 3.2 Edge 插件布局约定
插件统一使用：
- 顶部状态栏
- 中部核心功能区
- 底部快捷操作区

## 3.3 框图符号说明
- `┌─ ┐ └─ ┘`：表示容器
- `│`：表示竖向区域
- `├─┤`：表示区域切分
- 文本标签：表示该区域功能

# 7. 交互说明补充

## 7.1 页面原型不是视觉定稿
前端开发时可以根据技术栈调整样式，但结构和功能区域尽量保持一致。

## 7.2 推荐先做“后台功能通路”
优先让以下链路可用：
- 项目 → 文档上传 → 解析结果 → 生成测试 → 执行 → 报告

## 7.3 推荐插件先做“辅助能力”
优先让以下链路可用：
- 插件首页 → 当前页摘要 → 元素拾取 → 候选定位器
