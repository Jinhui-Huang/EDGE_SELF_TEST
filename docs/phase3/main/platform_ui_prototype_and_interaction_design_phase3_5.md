# 2. 设计定位

## 2.1 后台主平台
后台主平台是整个系统的**主操作中心**。

负责：
- 项目管理
- 测试文档管理
- 测试用例管理
- DSL / 状态机查看与编辑
- 执行控制
- 报告查看
- 模型配置
- 数据源配置
- DB 数据模板配置
- 审计与历史查看

## 2.2 Edge 插件辅助入口
Edge 插件不是主平台，而是**页面侧辅助入口**。

负责：
- 当前页摘要查看
- 页面元素拾取
- 候选定位器查看
- 当前运行状态快速查看
- 当前页快速调试/快速启动测试

# 4. 设计原则

## 4.1 后台主平台承载复杂流程
复杂的：
- 配置
- 文档管理
- 状态机查看
- 报告查看
- DB 差异查看  
都必须在后台主平台完成。

## 4.2 Edge 插件保持轻量
插件面板应尽量聚焦：
- 当前页
- 当前状态
- 当前定位
- 当前快速动作

## 4.3 页面分阶段建设
不要求一开始所有页面都齐全。  
应按 Phase 3 ~ 5 逐步扩展。

## 4.4 页面必须服务于“文档驱动自动测试”
不是为了好看而加页面，而是围绕主流程：
- 上传文档
- AI 解析
- 生成测试
- 执行测试
- 查看结果
- 查看数据变化与恢复

### 目标
查看当前页面被系统识别到的结构。

### 主要内容

- 表单字段摘要
- 按钮摘要

## 8.3 元素拾取面板
### 目标

## 8.4 候选定位器面板
### 目标

## 8.6 快速启动测试面板

# 17. 后续 UI 视觉设计建议

本阶段建议优先做**低保真功能原型**，不必一开始做重视觉稿。

## 视觉风格建议
- 企业后台风格
- 以信息密度和可读性优先
- 数据对比和风险提醒要明显
- 运行时页面要突出“当前状态”和“当前风险”

## 推荐视觉优先级
1. 信息结构清晰
2. 执行状态明确
3. 风险/失败/恢复警告醒目
4. 复杂配置页保持可折叠
5. 插件保持轻量简洁
